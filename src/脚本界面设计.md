# 脚本界面设计

本设计文档用于统一“酒馆助手脚本”的可见界面挂载与定位方式，目标是：

- 界面始终挂在酒馆主页面（父页面），不受脚本 iframe 尺寸限制
- 默认顶部居中，滚动页面不漂移
- 兼容电脑端与移动端
- 支持 Vue 挂载，并能正确把样式复制到父页面
- **界面可用性要求**：杜绝内容溢出无滚动、弹窗位置错误、功能堆砌
- **累赘说明**：功能验证期允许保留但需标记，最终交付期必须删除

> 约定：本文档用于“脚本项目”（`src/xxx` 仅有 `index.ts`）。前端界面项目（有 `index.html`）遵循其自身的 iframe 适配规则。

---

## 1. 核心原则（必须遵守）

### 1.1 挂载到父页面，而不是脚本 iframe

脚本在 iframe 内运行，但我们要让 UI 出现在酒馆主页面上层，因此：

- UI 容器必须 append 到父页面 `body`
- UI 样式必须注入/复制到父页面 `head`

推荐获取父页面 document：

```ts
const parentDoc = window.parent?.document ?? document;
```

推荐挂载位置：

```ts
const $mount = createScriptIdDiv().appendTo($('body', parentDoc));
```

样式复制（必须做，否则挂在父页面的 Vue 组件样式不会生效）：

```ts
const styleDestroy = teleportStyle($('head', parentDoc));
```

### 1.2 事件必须可观测（避免“点了没反应”）

脚本按钮点击后“无反应”通常只有两类原因：

- 事件没绑定成功 / 绑定到错误 document 上（界面挂在 iframe 里导致父页面看不见）
- `open()` 抛错但没有 `errorCatched`，控制台也没有任何线索

因此建议：

- 所有按钮事件用 `errorCatched(open)` 包裹
- 脚本加载、打开、关闭都要 `console.info` + `toastr`（至少其一）

### 1.3 弹窗定位：强制居中，禁止固定在右侧

- **默认弹窗必须采用“顶部居中浮窗”或“屏幕居中弹窗”**，禁止将弹窗固定在右侧（如 `right: 0` 或 `right: 10px`）
- 如果界面需要侧边栏，必须通过明确的“侧边栏按钮”触发，而不是让主弹窗默认靠右
- 弹窗必须支持点击遮罩层关闭（`overlay` 绑定关闭事件）

### 1.4 内容滚动强制要求

- 任何内容区域（包括弹窗主体、列表、表格）只要内容可能超出容器，**必须设置 `overflow-y: auto` 或 `overflow: auto`**，并配合 `min-height: 0` 确保 flex 布局下滚动生效
- 禁止出现内容被截断且无滚动条的情况
- 弹窗本身必须设置 `max-height`，防止撑出视口

### 1.5 功能拆分：禁止将复杂功能全部堆砌在主界面

- 脚本界面默认只显示**最核心的功能**（如一个主操作区）
- 次要功能、配置项、高级选项必须通过**次级弹窗、折叠面板（accordion）、选项卡（tabs）或下拉菜单**展开，避免主界面臃肿
- 如果一个界面的按钮/控件超过 **8 个** 且未做分组，视为违规

### 1.6 累赘说明的处理策略（分阶段）

为了兼顾开发调试与最终产品体验，采用**两阶段策略**：

#### 阶段一：功能验证期（初期）
- **允许**在界面上添加教学式说明文字，例如“示例：点击这里”“这是一个演示”“请输入内容（例如…）”等。
- 所有此类**临时说明**必须使用统一的 CSS 类名（如 `temp-note` 或 `dev-only`）包裹，以便后期批量删除。
- 建议在说明文字前加上 `[TEMP]` 前缀，方便搜索定位。

```html
<!-- 示例：临时说明 -->
<div class="temp-note">[TEMP] 这是一个示例按钮，功能实现后可删除此提示</div>
```

#### 阶段二：产品交付期（最终）
- **必须删除所有带有 `temp-note` / `dev-only` / `[TEMP]` 标记的累赘说明**。
- 界面只保留必要的功能标签、输入框占位符（仅限“请输入名称”这类格式提示），不得有任何教学式文案。
- 最终代码需通过第5节检查清单中的“无累赘说明”项。

> 开发建议：在功能确认后，使用 IDE 全局搜索 `temp-note`、`[TEMP]`、`dev-only` 等关键词，一键删除所有临时说明块。

### 1.7 尺寸采用“百分比 + 最大值”双保险

推荐默认值（可按界面复杂度微调）：

- `width: 95%`
- `max-width: 600px`（复杂表格类可提高到 `1000px`）
- `max-height: 80vh`
- 内容区内部滚动：`overflow-y: auto` + `min-height: 0`

### 1.8 必须带移动端媒体查询

推荐断点：`@media (max-width: 768px)`，移动端可做：

- `top` 略小（例如 `45px`）
- `max-width` 更窄（例如 `480px`）
- `max-height` 略大（例如 `85vh`）
- 触控区域优化：按钮高度 `>= 40px`

### 1.9 颜色与可读性（推荐用变量，避免硬编码）

脚本 UI 会叠在酒馆页面之上，配色建议遵循：

- 文本/边框/背景尽量用一组变量统一控制（便于换色与主题兼容）
- `primary`/强调色只用于主按钮与聚焦描边（避免花）
- 移动端需要更大的触控区域：按钮高度建议 `>= 40px`

推荐最小变量集合（可放在根容器上，如 `.th-script-modal`）：

```css
.th-script-modal {
  --th-surface: #151a23;
  --th-surface-2: #0f131b;
  --th-text: #e6e8ee;
  --th-muted: rgba(230, 232, 238, 0.68);
  --th-border: rgba(230, 232, 238, 0.12);
  --th-border-2: rgba(230, 232, 238, 0.08);
  --th-accent: #f59e0b;
  --th-accent-2: rgba(245, 158, 11, 0.22);
  --th-danger: #ff6b6b;
}
```

## 2. 两种挂载模式（如何选择）

### 2.1 父页面挂载（推荐默认）

适用：

- 悬浮窗、工具面板、弹层
- 需要叠在酒馆页面之上

注意：

- 需要 `teleportStyle($('head', parentDoc))` 复制样式
- 为避免与酒馆页面类名冲突，父页面挂载场景下**不要使用 tailwindcss**，尽量用组件内 `<style scoped>`

### 2.2 iframe 隔离挂载（仅在需要强隔离时使用）

适用：

- 界面希望与酒馆现有样式完全隔离
- 需要大量使用 tailwindcss class

实现方式参考 `util/script.ts` 的 `createScriptIdIframe()`，在 iframe `load` 后把样式注入到 iframe 的 `head`，并把 Vue mount 到 iframe `body`。

## 3. 标准 UI 结构（CSS 模板）

推荐所有脚本复用以下类名结构（避免散落的随意类名；也便于统一调试）。

## 4. 标准生命周期（Vue + 脚本）

下面是一份“可直接复用”的脚本侧模板（父页面挂载）。

```ts
import { createScriptIdDiv, teleportStyle } from '@util/script';
import { createApp } from 'vue';
import { createPinia } from 'pinia';
import App from './App.vue';

$(() => {
  const SCRIPT_NAME = '你的脚本名';
  appendInexistentScriptButtons([{ name: SCRIPT_NAME, visible: true }]);

  let app: ReturnType<typeof createApp> | null = null;
  let $container: JQuery | null = null;
  let styleDestroy: { destroy: () => void } | null = null;

  function open() {
    close();

    const parentDoc = window.parent?.document ?? document;

    $container = createScriptIdDiv().appendTo($('body', parentDoc));

    app = createApp(App).use(createPinia());
    (window as any).__myModalClose = close;
    app.mount($container[0]);

    styleDestroy = teleportStyle($('head', parentDoc));
    console.info(`[${SCRIPT_NAME}] 已打开界面`);
  }

  function close() {
    if (app) {
      app.unmount();
      app = null;
    }
    if ($container) {
      $container.remove();
      $container = null;
    }
    if (styleDestroy) {
      styleDestroy.destroy();
      styleDestroy = null;
    }
    delete (window as any).__myModalClose;
    console.info(`[${SCRIPT_NAME}] 已关闭界面`);
  }

  // 必须在 pagehide 清理，避免切换页面/聊天后残留
  $(window).on('pagehide', () => close());

  // 你自己的按钮/事件触发 open()
  eventOn(getButtonEvent(SCRIPT_NAME), errorCatched(open));
  toastr.success(`${SCRIPT_NAME} 脚本已加载`);
});
```

## 5. 检查清单（提交前自查）

- 容器是否 append 到父页面 `body`（而非 iframe body）
- 是否把样式复制/注入到父页面 `head`（`teleportStyle($('head', parentDoc))`）
- 是否使用顶部居中定位（`left: 50%` + `translateX(-50%)`）或屏幕居中弹窗
- **是否禁止了弹窗固定在右侧**
- 是否使用 `width: 95% + max-width + max-height`，并让内容区内部滚动
- **所有可能溢出的容器是否设置了 `overflow-y: auto` 并配合 `min-height: 0`**
- **界面功能是否过度堆砌（超过8个控件且未分组）**；复杂功能是否通过次级弹窗/折叠面板/选项卡拆分
- **累赘说明处理**：
  - 功能验证期：允许存在，但必须用 `temp-note` / `[TEMP]` 标记
  - 产品交付期：**必须删除所有标记过的说明文字**，且不得有任何教学式文案
- 是否包含 `@media (max-width: 768px)` 移动端修正
- 是否在 `pagehide` 里卸载 Vue、移除容器、销毁样式复制
- 按钮触控区域在移动端是否足够大（高度 ≥ 40px）
